Introduction to Terraform 
 

What is Terraform 

Terraform is an Infrastructure as Code (IaC) tool used to provision and manage infrastructure automatically. 

It allows engineers to define infrastructure using code instead of manual configuration. 

Terraform was developed by HashiCorp. 

Using Terraform we can create: 

Virtual machines 

Networks 

Kubernetes clusters 

Databases 

Storage systems 

Load balancers 

 

Why Terraform is Important 

Before Infrastructure as Code, infrastructure was managed manually. 

Example: 

Engineers had to manually create resources in cloud consoles such as: 

Amazon Web Services 

Microsoft Azure 

Google Cloud 

Problems with manual infrastructure management: 

configuration errors 

lack of reproducibility 

difficult environment replication 

slow provisioning 

Terraform solves these problems by allowing infrastructure to be defined, versioned, and automated using code. 

 

2. What is Infrastructure as Code (IaC) 

Infrastructure as Code is the practice of managing infrastructure using configuration files. 

Instead of manually creating servers, we define them in code. 

Example Terraform configuration: 

 

resource "aws_instance" "web" { 

  ami           = "ami-123456" 

  instance_type = "t2.micro" 

} 

 

This code tells Terraform to create an EC2 instance. 

Benefits of IaC: 

automation 

repeatability 

version control 

faster deployments 

 

3. Terraform Architecture 

Terraform works using several components. 

Terraform CLI 

Command-line interface used to run Terraform commands. 

Examples: 

 

terraform init 

terraform plan 

terraform apply 

 

 

Providers 

Providers allow Terraform to interact with cloud platforms. 

Examples: 

AWS Provider 

Azure Provider 

Google Cloud Provider 

Kubernetes Provider 

Example provider configuration: 

 

provider "aws" { 

  region = "us-east-1" 

} 

 

 

Resources 

Resources represent infrastructure components. 

Examples: 

EC2 instances 

VPC networks 

S3 buckets 

Load balancers 

Example: 

 

resource "aws_s3_bucket" "data" { 

  bucket = "company-data-bucket" 

} 

 

 

State File 

Terraform stores the infrastructure state in a file called: 

 

terraform.tfstate 

 

This file tracks: 

what resources exist 

their configuration 

their current status 

 

4. Terraform Workflow 

Terraform follows a standard workflow. 

Step 1 Initialize 

Initialize the Terraform project. 

 

terraform init 

 

This downloads required providers. 

 

Step 2 Plan 

Terraform generates an execution plan. 

 

terraform plan 

 

This shows what infrastructure will be created or modified. 

Example output: 

 

+ aws_instance.web will be created 

 

 

Step 3 Apply 

Apply the configuration to create infrastructure. 

 

terraform apply 

 

Terraform will provision resources automatically. 

 

Step 4 Destroy 

Destroy infrastructure when no longer needed. 

 

terraform destroy 

 

 

5. Terraform Configuration Language 

Terraform uses HCL (HashiCorp Configuration Language). 

HCL is designed to be human readable and easy to write. 

Example structure: 

 

provider 

resource 

variables 

output 

modules 

 

 

6. Variables in Terraform 

Variables make Terraform configurations flexible. 

Example: 

 

variable "instance_type" { 

  default = "t2.micro" 

} 

 

Use variable: 

 

instance_type = var.instance_type 

 

Benefits: 

reusable configurations 

environment specific values 

 

7. Terraform Modules 

Modules allow reusable infrastructure code. 

Example: 

Instead of writing the same configuration repeatedly, we create modules. 

Example module structure: 

 

modules/ 

   vpc/ 

   ec2/ 

   rds/ 

 

Usage: 

 

module "vpc" { 

  source = "./modules/vpc" 

} 

 

Benefits: 

reusability 

standardization 

easier maintenance 

 

8. Terraform State Management 

Terraform state is critical. 

Default state file: 

 

terraform.tfstate 

 

Problems with local state: 

difficult for teams 

risk of conflicts 

Best practice: 

Store state remotely. 

Example remote backends: 

**Amazon Web Services S3 

Azure Storage 

Terraform Cloud 

Example remote state: 

 

backend "s3" { 

  bucket = "terraform-state" 

  key    = "prod/terraform.tfstate" 

  region = "us-east-1" 

} 

 

 

9. Terraform Best Practices 

Some important best practices: 

Use version control (Git) 

Use remote state storage 

Separate environments 

Example: 

 

dev 

stage 

prod 

 

Use modules for reusable infrastructure. 

Implement proper security policies.

Terraform Sample Script to Provision AWS EC2 Instance 
 
1. Provider Configuration 

provider "aws" { region = "ap-south-1" # Mumbai region } 

2. Key Pair (Optional but recommended) 

resource "aws_key_pair" "deployer" { key_name = "ec2-key" public_key = file("~/.ssh/id_rsa.pub") } 

3. Security Group (Allow SSH + HTTP) 

resource "aws_security_group" "ec2_sg" { name = "ec2-security-group" description = "Allow SSH and HTTP" 

ingress { description = "SSH" from_port = 22 to_port = 22 protocol = "tcp" cidr_blocks = ["0.0.0.0/0"] } 

ingress { description = "HTTP" from_port = 80 to_port = 80 protocol = "tcp" cidr_blocks = ["0.0.0.0/0"] } 

egress { from_port = 0 to_port = 0 protocol = "-1" cidr_blocks = ["0.0.0.0/0"] } } 

4. EC2 Instance 

resource "aws_instance" "my_ec2" { ami = "ami-0f58b397bc5c1f2e8" # Amazon Linux 2 (Mumbai) instance_type = "t2.micro" 

key_name = aws_key_pair.deployer.key_name security_groups = [aws_security_group.ec2_sg.name] 

tags = { Name = "MyTerraformEC2" } } 

5. Output Public IP 

output "instance_public_ip" { value = aws_instance.my_ec2.public_ip }
