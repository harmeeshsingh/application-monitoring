# main files

Core lab documentation and reference material. Each file covers a specific SRE/observability topic — theory, architecture, and implementation steps.

> **Note:** Files without extensions are plain-text lab guides. Open them in any text editor or GitHub's file viewer.

| `Dynatrace_Observability` | Observability | Using Dynatrace for full-stack observability and anomaly detection |
| `EKS-self-healing script-Toil Reduction and Self-Healing Systems` | EKS / SRE | Automating pod recovery on EKS to reduce operational toil |

| `Log_Correlation_for_Debugging_ELK_Stack` | ELK / Logging | Correlating logs across services using Elasticsearch, Logstash, Kibana |
| `Observability_for_Microservices_Applications.txt` | Microservices | Implementing metrics, logs, and traces for microservice architectures |
| `Python Script to Check System Logs - Toil Reduction and Self-Healing Systems` | Toil Reduction | Python script to parse system logs and trigger automated remediation |
| `Security Incident Simulation - Incident Management` | Security | Simulating security incidents and practicing response workflows |
| `active-passive setups -Incident Management and Fault Tolerance` | Fault Tolerance | Active-passive architecture setup for high availability and failover |
